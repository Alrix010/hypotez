# Module: g4f/typing.py

## Overview

This module defines custom typing annotations for use within the `gpt4free` module of the `hypotez` project. It primarily focuses on providing type hints for functions, methods, and data structures related to interactions with the `gpt4free` API.

## Details

The module aims to enhance code readability and maintainability by specifying precise data types for variables, function arguments, and return values. It utilizes Python's built-in typing features as well as custom type aliases for specific data structures relevant to GPT-4Free interactions.

## Classes

This module does not contain any classes.

## Functions

There are no functions defined in this module.

## Custom Type Aliases

The following custom type aliases are defined within this module:

### `SHA256`

**Description:** This type alias represents a SHA256 hash, which is a string. It enhances code clarity by explicitly indicating the data type for SHA256 hashes.

**Example:**

```python
sha256_hash: SHA256 = 'e567314d44315a60b72f907972d064f4e21556f400c9c9449893796858a4819b'
```

### `CreateResult`

**Description:** This type alias represents an iterator that yields either a string or a `ResponseType` object. It facilitates working with results generated by the `gpt4free` API, which can be either text or a structured response object.

**Example:**

```python
def create_content() -> CreateResult:
    """Generates text or response objects."""
    yield "Some text"
    yield ResponseType(text="Another response")

for result in create_content():
    if isinstance(result, str):
        print(f"Text: {result}")
    elif isinstance(result, ResponseType):
        print(f"Response: {result.text}")
```

### `AsyncResult`

**Description:** This type alias represents an asynchronous iterator that yields either a string or a `ResponseType` object. It is similar to `CreateResult` but designed for asynchronous operations.

**Example:**

```python
async def async_create_content() -> AsyncResult:
    """Asynchronously generates text or response objects."""
    yield "Some text"
    yield ResponseType(text="Another response")

async for result in async_create_content():
    if isinstance(result, str):
        print(f"Text: {result}")
    elif isinstance(result, ResponseType):
        print(f"Response: {result.text}")
```

### `Messages`

**Description:** This type alias represents a list of message objects. Each message object is a dictionary containing fields such as "role", "content", and potentially other metadata. This data structure is used to represent conversation histories with AI models.

**Example:**

```python
messages: Messages = [
    {"role": "user", "content": "Hello!"},
    {"role": "assistant", "content": "Hi there!"},
]
```

### `Cookies`

**Description:** This type alias represents a dictionary of cookies. Cookies are used for managing user sessions and preferences on websites.

**Example:**

```python
cookies: Cookies = {
    "session_id": "1234567890",
    "user_id": "abc123",
}
```

### `Image`

**Description:** This type alias represents an image. It can be a PIL Image object, a path to an image file, or a bytes-like object representing the image data.

**Example:**

```python
from PIL.Image import Image

image: Image = Image.open("image.png")
```

### `ImageType`

**Description:** This type alias represents a more general type for image data. It can be a string (representing a path to an image file), bytes-like object, an `IO` object representing a file, a PIL Image object, or a path-like object.

**Example:**

```python
image: ImageType = "image.png"  # Path to an image file
image: ImageType = open("image.jpg", "rb").read()  # Bytes-like object
image: ImageType = open("image.gif", "rb")  # IO object
```

### `MediaListType`

**Description:** This type alias represents a list of media items, each containing an image and an optional caption.

**Example:**

```python
media_list: MediaListType = [
    ("image.png", "Beautiful landscape"),
    (open("photo.jpg", "rb"), "A stunning sunset"),
]
```
