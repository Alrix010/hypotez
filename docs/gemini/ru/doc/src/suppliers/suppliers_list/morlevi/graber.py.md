# Модуль для сбора данных о товарах с сайта morlevi.co.il

## Обзор

Модуль предназначен для сбора информации о товарах с сайта `morlevi.co.il`. Он наследуется от базового класса `Graber` и переопределяет его методы для обработки специфичных полей и элементов на страницах товаров `morlevi.co.il`.

## Подробней

Этот модуль предназначен для автоматизированного сбора данных о товарах с сайта `morlevi.co.il`. Он расширяет функциональность базового класса `Graber`, адаптируя методы для работы с уникальной структурой страниц товаров `morlevi.co.il`.

## Классы

### `Graber`

**Описание**: Класс `Graber` предназначен для сбора информации о товарах с сайта `morlevi.co.il`. Он наследуется от базового класса `Graber` (`Grbr`) и переопределяет его методы для обработки специфичных полей и элементов на страницах товаров `morlevi.co.il`.

**Наследует**:

- `src.suppliers.graber.Graber`

**Атрибуты**:

- `supplier_prefix` (str): Префикс поставщика, установлен в `morlevi`.
- `driver` (Optional['Driver']): Экземпляр веб-драйвера для управления браузером.
- `lang_index` (Optional[int]): Индекс языка, используемый для локализации.

**Методы**:
- `__init__`: Инициализирует экземпляр класса `Graber`.

### Методы класса

**Назначение**: Инициализирует экземпляр класса `Graber`, вызывая конструктор родительского класса `Graber` (`Grbr`) и устанавливая специфичные для `morlevi` параметры конфигурации.

**Как работает функция**:

1.  Вызывает конструктор родительского класса `Graber` (`Grbr`) с передачей префикса поставщика (`supplier_prefix`), экземпляра веб-драйвера (`driver`) и индекса языка (`lang_index`).
2.  Устанавливает драйвер в конфигурации (`Config.driver`) для использования в других частях класса.
3.  Устанавливает локатор для декоратора (`Config.locator_for_decorator`) из атрибута `close_pop_up` объекта `product_locator`.
