### Как использовать этот блок кода
=========================================================================================

Описание
-------------------------
Данный код определяет абстрактный класс `TinyTool`, который служит базовым классом для создания различных инструментов, используемых агентами в системе. Он включает в себя логику инициализации, защиты от реальных побочных эффектов, проверки владения инструментом и обработки действий.

Шаги выполнения
-------------------------
1. **Инициализация инструмента**:
   - При создании экземпляра `TinyTool` необходимо указать имя (`name`), описание (`description`), владельца (`owner`), флаг наличия реальных побочных эффектов (`real_world_side_effects`), экспортер (`exporter`) и обогатитель (`enricher`).
   - Функция `__init__` сохраняет переданные параметры в атрибуты экземпляра класса.

2. **Защита от реальных побочных эффектов**:
   - Метод `_protect_real_world` проверяет, имеет ли инструмент реальные побочные эффекты. Если да, то выводит предупреждение в логи.

3. **Проверка владения инструментом**:
   - Метод `_enforce_ownership` проверяет, имеет ли агент право на использование данного инструмента. Если инструмент имеет владельца и агент не является этим владельцем, выбрасывается исключение `ValueError`.

4. **Обработка действия**:
   - Метод `process_action` выполняет следующие шаги:
     - Вызывает метод `_protect_real_world` для проверки реальных побочных эффектов.
     - Вызывает метод `_enforce_ownership` для проверки владения инструментом.
     - Вызывает абстрактный метод `_process_action` для выполнения конкретного действия, специфичного для подкласса.

5. **Абстрактные методы**:
   - Метод `_process_action` должен быть реализован в подклассах для выполнения конкретного действия инструмента.
   - Методы `actions_definitions_prompt` и `actions_constraints_prompt` также должны быть реализованы в подклассах для предоставления информации о возможных действиях и ограничениях инструмента.

Пример использования
-------------------------

```python
from tinytroupe.tools import logger
from tinytroupe.tools.tiny_tool import TinyTool

class MyTool(TinyTool):
    def __init__(self, name, description, owner=None, real_world_side_effects=False, exporter=None, enricher=None):
        super().__init__(name, description, owner, real_world_side_effects, exporter, enricher)

    def _process_action(self, agent, action: dict) -> bool:
        """
        Выполняет действие инструмента.

        Args:
            agent: Агент, выполняющий действие.
            action (dict): Словарь с параметрами действия.

        Returns:
            bool: True, если действие выполнено успешно, False в противном случае.

        """
        logger.info(f"Агент {agent.name} выполняет действие {action} с инструментом {self.name}")
        # Здесь должна быть логика выполнения действия
        return True

    def actions_definitions_prompt(self) -> str:
        """
        Определяет prompt для описания возможных действий инструмента.

        Returns:
            str: Prompt с описанием возможных действий.

        """
        return "Описание возможных действий инструмента."

    def actions_constraints_prompt(self) -> str:
        """
        Определяет prompt для ограничений инструмента.

        Returns:
            str: Prompt с описанием ограничений инструмента.

        """
        return "Описание ограничений инструмента."
