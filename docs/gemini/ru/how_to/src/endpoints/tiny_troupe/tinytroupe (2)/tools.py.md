## Как использовать TinyTool и его подклассы в TinyTroupe

=========================================================================================

### Описание

`TinyTool` - это абстрактный класс, который служит шаблоном для создания специализированных инструментов, доступных агентам в TinyTroupe. Он определяет общий интерфейс для взаимодействия с агентами и предоставляет базовую функциональность, такую как проверка прав доступа и предупреждение о потенциальных побочных эффектах в реальном мире.

### Шаги выполнения

1. **Создание инструмента:** 
   - Наследуйтесь от `TinyTool`, чтобы создать новый тип инструмента.
   - Определите имя, описание, владельца (необязательно), потенциальные побочные эффекты в реальном мире, а также экспортер и обогатитель (необязательно).
   - Реализуйте метод `_process_action`, который обрабатывает действия, которые агент может выполнять с помощью этого инструмента.

2. **Определение действий:** 
   - Предоставьте описание действий, доступных для данного инструмента, в методе `actions_definitions_prompt`.
   - Используйте форматирование Markdown для структурирования и оформления информации.

3. **Определение ограничений:**
   - Определите ограничения и правила для использования инструмента в методе `actions_constraints_prompt`.

4. **Использование инструмента:**
   - Агент вызывает метод `process_action` инструмента, передавая действие, которое он хочет выполнить.
   - `TinyTool` автоматически проверяет права доступа агента и предупреждает о потенциальных последствиях в реальном мире.

5. **Обработка действий:**
   - Метод `_process_action` инструмента реализует логику, необходимую для обработки действия.
   - В случае необходимости, инструмент может использовать экспортер и обогатитель для обработки результатов.

### Пример использования

```python
from tinytroupe.tools import TinyTool, TinyWordProcessor

# Создаем новый инструмент "Word Processor"
word_processor = TinyWordProcessor(owner="Alice", exporter=ArtifactExporter(), enricher=TinyEnricher())

# Агент "Bob" пытается использовать инструмент "Word Processor"
agent_bob = ...  # Предположим, агент Bob уже определен

# Агент Bob хочет написать документ
action = {
    "type": "WRITE_DOCUMENT",
    "content": {
        "title": "My First Document",
        "content": "This is the content of my document.",
        "author": "Bob"
    }
}

# Агент Bob отправляет действие инструменту
word_processor.process_action(agent_bob, action)

# Инструмент "Word Processor" проверяет права доступа, 
# обрабатывает действие и создает документ
```

### Примечания

- Класс `TinyTool` предоставляет базовый каркас для создания специализированных инструментов.
- Метод `_process_action`  реализует логику обработки действия.
- Методы `actions_definitions_prompt` и `actions_constraints_prompt` описывают действия и ограничения.
- Инструмент автоматически проверяет права доступа и предупреждает о потенциальных последствиях в реальном мире.
